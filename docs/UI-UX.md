# Cloudflare Tunnel Manager - UI/UX 设计文档

## 1. 设计理念

### 1.1 设计目标

- **简洁直观**：用户无需了解 Cloudflare 即可上手
- **状态清晰**：一眼可见 Tunnel 状态
- **快速响应**：所有操作即时反馈

### 1.2 视觉风格

- **现代简约**：扁平化设计，少即是多
- **状态导向**：用颜色区分状态（绿=运行中，红=错误，灰=停止）
- **暗色优先**：默认暗色主题，兼顾护眼

## 4. 组件规范

### 4.1 按钮

| 类型 | 用途 | 样式 |
|------|------|------|
| Primary | 主要操作（启动、保存） | 蓝色背景，白色文字 |
| Secondary | 次要操作（取消、编辑） | 灰色边框，无填充 |
| Danger | 危险操作（删除、停止） | 红色背景/文字 |
| Ghost | 辅助操作（查看日志） | 无边框，悬停显示 |

### 4.2 状态指示

```
● 运行中 (Running)    → 绿色圆点 + "Running"
● 已停止 (Stopped)    → 灰色圆点 + "Stopped"
● 错误 (Error)        → 红色圆点 + "Error"
● 启动中 (Starting)   → 黄色圆点 + 加载动画
```

### 4.3 图标规范

使用 **Lucide Icons** 或 **Heroicons**：

| 功能 | 图标 |
|------|------|
| 添加 | `plus` |
| 编辑 | `pencil` |
| 删除 | `trash-2` |
| 启动 | `play` |
| 停止 | `square` |
| 设置 | `settings` |
| 日志 | `file-text` |
| 搜索 | `search` |

## 5. 交互流程

### 5.1 首次使用

```
1. 打开应用
      ↓
2. 显示空状态页面 + "添加您的第一个 Tunnel"
      ↓
3. 用户点击 [New Tunnel]
      ↓
4. 弹出对话框，提示如何获取 Token
      ↓
5. 用户输入名称 + Token → [保存并启动]
      ↓
6. 自动启动 Tunnel，显示运行状态
```

### 5.2 启动 Tunnel

```
1. 用户点击 [Start] 按钮
      ↓
2. 按钮变为 [Starting...] + 禁用状态
      ↓
3. 后端启动 cloudflared 进程
      ↓
4. 实时日志显示在 Logs 页面
      ↓
5. 状态变为 "Running" + 绿色指示灯
      ↓
6. 显示连接信息（如 tunnel URL）
```

## 6. 响应式设计

### 6.1 断点

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| Mobile | < 640px | 单列，卡片全宽 |
| Tablet | 640-1024px | 双列网格 |
| Desktop | > 1024px | 单列，固定宽度 800px |

### 6.2 系统托盘

- **托盘图标**：显示 Tunnel 状态（绿/灰/红）
- **右键菜单**：
  - 显示/隐藏窗口
  - 快速启动/停止
  - 退出应用

| 场景 时 | 动画 |长 |

| 页面切换 | 淡入 | 150ms |

| 状态变化 | 颜色过渡 | 200ms |
| 对话框 | 缩放 + 淡入 | 200ms |
| 列表项 | 淡入 + 上移 | 300ms |

## 8. 无障碍设计

- **键盘导航**：Tab 键可遍历所有可交互元素
- **焦点指示**：可见的焦点轮廓
- **颜色对比**：文字与背景对比度 ≥ 4.5:1
- **屏幕阅读器**：所有图标有 aria-label
