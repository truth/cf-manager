目前市面上**还没有一个非常成熟、通用的开源 Cloudflare Tunnel (cloudflared) GUI 客户端**，特别是基于 **Rust + Tauri** 的。

绝大多数用户通过 CLI 命令行或者 Cloudflare 网页端的 Zero Trust Dashboard 来配置。

这其实是一个**非常棒的开源项目点子**。如果你想自己开发，这是一个极佳的练手项目，且非常有实用价值（对于不喜欢敲命令的用户）。

以下是如何使用 **Rust + Tauri** 封装 `cloudflared` 的完整技术架构和实现指南。

---

### 1. 核心架构设计：Sidecar 模式

Tauri 有一个专门的功能叫 **Sidecar (边车)**，专门用于在 Tauri 应用中打包和调用外部二进制文件（如 Python 脚本、Go 程序、或者这里的 `cloudflared`）。

**架构图：**

```mermaid
graph TD
    User[用户 UI] -->|点击开启| Frontend[Tauri Frontend (Vue/React)]
    Frontend -->|invoke('start_tunnel')| Rust[Rust Backend (main.rs)]
    Rust -->|tauri::shell::Command| Binary[("cloudflared.exe (Sidecar)")]
    Binary -->|建立连接| Cloudflare[Cloudflare Edge]
    Binary -.->|Stdout/Logs| Rust
    Rust -.->|Emit Event| Frontend

```

### 2. 开发步骤详解

#### 第一步：准备二进制文件 (Sidecar)

你需要下载对应平台的 `cloudflared` 二进制文件，并按照 Tauri 的命名规则重命名。

1. 在项目根目录创建 `src-tauri/binaries/` 文件夹。
2. 下载 `cloudflared` (Windows/Linux/macOS)。
3. **重命名规则**：`name-target-triple`。
* Windows: `cloudflared-x86_64-pc-windows-msvc.exe`
* macOS (M1): `cloudflared-aarch64-apple-darwin`
* Linux: `cloudflared-x86_64-unknown-linux-gnu`



#### 第二步：配置 `tauri.conf.json`

你需要明确告诉 Tauri 允许执行这个外部命令。

```json
{
  "tauri": {
    "bundle": {
      "externalBin": [
        "binaries/cloudflared" // 不需要加 .exe 或架构后缀，Tauri 会自动匹配
      ]
    },
    "allowlist": {
      "shell": {
        "all": false,
        "execute": true,
        "sidecar": true,
        "scope": [
          { "name": "binaries/cloudflared", "sidecar": true, "args": true }
        ]
      }
    }
  }
}

```

#### 第三步：Rust 后端实现 (核心逻辑)

在 `src-tauri/src/main.rs` 中，你需要编写逻辑来：

1. 启动 `cloudflared` 进程。
2. **持有进程句柄**（防止 Rust 函数结束进程就挂了）。
3. 实时读取日志并发送给前端。

```rust
use tauri::{
    command, AppHandle, Manager, Window,
};
use tauri::api::process::{Command, CommandEvent};
use std::sync::{Arc, Mutex};
use std::collections::HashMap;

// 用于存储运行中的子进程，以便后续可以杀死它
struct ChildProcesses(Arc<Mutex<HashMap<String, tauri::api::process::CommandChild>>>);

#[command]
fn start_tunnel(
    token: String, 
    window: Window, 
    state: tauri::State<ChildProcesses>
) -> String {
    // 使用 Sidecar 启动
    let (mut rx, child) = Command::new_sidecar("cloudflared")
        .expect("failed to create sidecar")
        // 传入参数: tunnel run --token <TOKEN>
        .args(&["tunnel", "run", "--token", &token]) 
        .spawn()
        .expect("Failed to spawn sidecar");

    // 保存子进程句柄
    state.0.lock().unwrap().insert("tunnel".to_string(), child);

    // 异步监听日志
    tauri::async_runtime::spawn(async move {
        while let Some(event) = rx.recv().await {
            if let CommandEvent::Stdout(line) = event {
                // 发送日志到前端
                window.emit("tunnel-log", line).unwrap();
            } else if let CommandEvent::Stderr(line) = event {
                 window.emit("tunnel-error", line).unwrap();
            }
        }
    });

    "Tunnel started".into()
}

#[command]
fn stop_tunnel(state: tauri::State<ChildProcesses>) -> String {
    let mut processes = state.0.lock().unwrap();
    if let Some(child) = processes.remove("tunnel") {
        child.kill().expect("Failed to kill tunnel");
        return "Tunnel stopped".into();
    }
    "No tunnel running".into()
}

fn main() {
    tauri::Builder::default()
        .manage(ChildProcesses(Arc::new(Mutex::new(HashMap::new()))))
        .invoke_handler(tauri::generate_handler![start_tunnel, stop_tunnel])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}

```

#### 第四步：前端调用 (Vue/React)

在前端，你只需要提供一个输入框让用户填 Token，然后调用 Rust。

```javascript
import { invoke } from '@tauri-apps/api/tauri';
import { listen } from '@tauri-apps/api/event';

// 启动
async function start() {
  await invoke('start_tunnel', { token: "eyJhIjoi..." });
}

// 监听日志
await listen('tunnel-log', (event) => {
  console.log('Log:', event.payload);
  // 更新到 UI 文本域
});

```

### 3. 需要解决的痛点 (User Experience)

如果你要做这个项目，以下是你在 UI 设计上需要优化的点：

1. **Token 获取引导**：
普通用户不知道去哪里找 Token。你可以在 UI 上做一个教程：*“请去 Cloudflare Zero Trust 面板 -> Access -> Tunnels -> Install，复制其中的 Token 字符串”*。
2. **配置持久化**：
使用 `tauri-plugin-store` 把用户的 Token 存在本地，下次打开自动填入。
3. **多路复用配置 (Advanced)**：
`cloudflared` 其实支持本地配置文件 (`config.yml`)。如果你想做一个高级版 UI，可以提供一个图形化界面来生成这个 YAML 文件（比如添加多个 `ingress` 规则），而不是仅仅运行 `tunnel run`。
4. **状态指示灯**：
通过解析日志中的 "Connection Registered" 关键词，在 UI 上把红灯变绿灯。

### 总结

目前确实没有一个知名的开源 Tauri 项目做这个。**这是一个很好的机会**。

利用 **Rust 的安全性** 处理进程管理，利用 **Tauri 的 Sidecar** 机制打包官方二进制文件，你可以很快（大约 1-2 天）写出一个 MVP（最小可行性产品），不仅能解决你自己的需求，发布到 GitHub 上可能还会收获不少 Star。
